# Cash Register — Project Form Data

**Title:** Cash Register

**Headline:** Mobile-first POS for restaurants and bars.

**Release Date:** March - 2025

**Link:** https://cash-register-a85839.gitlab.io/

## Description
Cash Register was born out of necessity during a rural event I helped organize. The challenge of building a functional POS system in under 8 hours was intense but incredibly educational. Working under time pressure forced me to make quick architectural decisions and prioritize features that would actually be used.

The iterative development during the event itself was a unique experience - I was able to see real users interact with the system and make improvements on the fly. The floating cart feature for mobile was a direct result of watching people struggle with the initial design.

This project taught me the value of simplicity, rapid iteration, and building for real-world constraints. Using Supabase for real-time updates was crucial, and seeing orders appear instantly across multiple devices was satisfying. It's a great example of how constraints can lead to creative solutions.

## Achievements
- Built a working mobile-first POS in under 8 hours and iterated live during the event.
- Created a catalog split into drink/food categories with per-product pricing.
- Added a floating mobile cart for fast jumps between the last added item and checkout.
- Delivered real-time order history with timestamps and live updates via Supabase.
