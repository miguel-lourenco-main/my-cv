# O Guardanapo — Project Form Data

**Title:** O Guardanapo

**Headline:** Web application for the snack-bar O Guardanapo, built as a Turborepo monorepo with Next.js, TypeScript, Supabase, and a shared UI/infra toolkit.

**Release Date:** January - 2025

**Link:** https://o-guardanapo.site

## Description
Working on O Guardanapo was a deeply personal and educational experience for me. My parents run a snack-bar in the heart of Lisbon, and this was my first time building a fully fledged web application for real users rather than just personal use.

I created the site to improve how my mom creates and publishes the daily menu and to make it easier for my dad to communicate with foreign customers who often struggle to order and end up playing 'charades' at the counter. Along the way I learned hard lessons about choosing the right tools—starting from an overpowered template with Stripe, authentication and many other features when all I truly needed was Supabase added a lot of unnecessary complexity and refactoring work.

The final product lets my mom manage a multilingual daily menu on the homepage, customers explore each dish with photos, ingredients and allergen or dietary labels, and anyone quickly access the menu through QR codes around the snack-bar. Having my parents as direct users gave me fast feedback and pushed me to think about planning, UI design, infrastructure, deployment and iterative refinement from both the business-owner and customer perspectives.

## Achievements
- Shipped my first production web app for real users (my parents' snack-bar in Lisbon).
- Built a multilingual daily-menu manager with photos, ingredients, and allergen/dietary labels.
- Enabled quick menu access via QR codes placed around the snack-bar.
- Structured the project as a Turborepo + pnpm monorepo with shared UI, Supabase, routing, i18n, monitoring, and mailer packages.
